## 1. IPD 流程概述

### 1.1 什么是 IPD

IPD (Integrated Product Development) 是一种集成产品开发流程，强调：
- **跨职能协作**: 研发、市场、服务、测试等团队紧密合作
- **阶段性门控**: 每个阶段有明确的交付物和评审点
- **异步开发与同步验证**: 开发与验证并行，缩短周期

### 4.3 开发规范

#### 编码规范

参见 [.agents/skills/code-review/references/custom-code-style.md](../../.agents/skills/code-review/references/custom-code-style.md)

#### Git 工作流

```bash
# 1. 创建功能分支
git checkout -b feature/new-feature

# 2. 开发并提交
git add .
git commit -m "feat: add new feature

- Feature description
- Implementation details
- Test cases"

# 3. 推送并创建 PR
git push origin feature/new-feature

# 4. Code Review 后合并
git checkout main
git merge feature/new-feature
git push origin main
```

#### Commit 规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

| Type | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档变更 |
| style | 代码格式 |
| refactor | 重构 |
| test | 测试相关 |
| perf | 性能优化 |
| chore | 构建/工具 |

### 4.4 持续集成

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Build
        run: |
          mkdir build && cd build
          cmake ..
          make -j$(nproc)
      
      - name: Unit Tests
        run: |
          cd build
          ctest --output-on-failure
          
      - name: Integration Tests
        run: |
          ./test_integration.sh
          
      - name: Performance Tests
        run: |
          ./test_performance.sh
```

### 8.2 文档 AI 友好规范

为使文档 AI 可理解和生成，遵循以下规范：

| 规范 | 说明 | 示例 |
|------|------|------|
| **结构化** | 使用明确的标题层级 | `## 1.`, `### 1.1.` |
| **代码示例** | 提供可运行的代码 | 完整的函数实现 |
| **类型明确** | 明确参数类型和返回值 | `int func(string input)` |
| **边界条件** | 说明边界情况 | `N > 0` |
| **错误处理** | 说明错误场景 | `失败返回 -1` |
| **可执行片段** | 关键逻辑可执行 | 伪代码算法 |
